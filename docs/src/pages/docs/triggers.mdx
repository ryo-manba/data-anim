# Triggers

data-anim supports four trigger types that control when an animation plays. Set them with the `data-anim-trigger` attribute.

## Viewport (Default)

The animation plays when the element scrolls into the viewport. This is the default behavior.

<CodeBlock
  code={`<!-- These are equivalent -->
<div data-anim="fadeInUp">Viewport trigger (implicit)</div>
<div data-anim="fadeInUp" data-anim-trigger="viewport">Viewport trigger (explicit)</div>`}
  lang="html"
/>

<AnimationDemo name="fadeInUp" trigger="viewport" />

### State Transitions

```
Hidden (below viewport) → Enters viewport → Animation plays → Visible
                                                              ↓
                                          (if once="false") Leaves viewport → Hidden again
```

### Offset

Use `data-anim-offset` to control how many pixels into the viewport the element must be before triggering:

<CodeBlock
  code={`<div data-anim="fadeIn" data-anim-offset="200">
  Triggers 200px after entering viewport
</div>`}
  lang="html"
/>

## Hover

Animation plays when the user hovers over the element. Great for interactive UI elements.

<CodeBlock
  code={`<div data-anim="pulse" data-anim-trigger="hover">
  Hover over me
</div>`}
  lang="html"
/>

<AnimationDemo name="pulse" trigger="hover" />

### Accessibility

Hover animations also respond to `focus-visible` for keyboard users, ensuring accessibility without extra work.

## Click

Animation plays when the user clicks the element.

<CodeBlock
  code={`<button data-anim="bounce" data-anim-trigger="click">
  Click me to bounce
</button>`}
  lang="html"
/>

<AnimationDemo name="bounce" trigger="click" />

### State Transitions

```
Idle → Click → Animation plays → Returns to idle
                                  ↓
                     (if once="true") Stays in final state
```

## Focus

Animation plays when the element receives focus. Ideal for form inputs.

<CodeBlock
  code={`<input
  type="text"
  data-anim="pulse"
  data-anim-trigger="focus"
  placeholder="Focus me"
/>`}
  lang="html"
/>

<AnimationDemo name="pulse" trigger="focus" />

### Accessibility Notes

- The focus trigger works with both mouse focus and keyboard focus (`Tab` key)
- Animations respect `prefers-reduced-motion` media query
- Use subtle animations (like `pulse` or `shake`) for focus rather than large movements

## Combining Triggers

Each element can only have one trigger. If you need multiple interactions, wrap the element:

<CodeBlock
  code={`<!-- Viewport trigger for entrance, hover for interaction -->
<div data-anim="fadeInUp" data-anim-trigger="viewport">
  <button data-anim="pulse" data-anim-trigger="hover">
    Interactive button
  </button>
</div>`}
  lang="html"
/>

## Summary

| Trigger | Attribute Value | Use Case |
|---------|----------------|----------|
| Viewport | `viewport` (default) | Scroll animations, page load reveals |
| Hover | `hover` | Buttons, cards, interactive elements |
| Click | `click` | Buttons, toggles, confirmations |
| Focus | `focus` | Form inputs, links, interactive widgets |
