# Triggers

data-anim supports four trigger types. Click each tab to see it in action.

<TriggerDemo />

## Viewport (Default)

The element animates when it scrolls into the viewport using `IntersectionObserver`.

<CodeBlock
  code={'<div data-anim="fadeInUp">\n  Animates on scroll (default)\n</div>\n\n<!-- With custom offset -->\n<div\n  data-anim="fadeIn"\n  data-anim-offset="200"\n>\n  Triggers 200px into viewport\n</div>'}
  lang="html"
/>

## Hover

Plays on `mouseenter`, resets on `mouseleave`. Also responds to `focus-visible` for keyboard users.

<CodeBlock
  code={'<div data-anim="pulse" data-anim-trigger="hover">\n  Hover over me\n</div>'}
  lang="html"
/>

## Click

Replays on every click. Great for buttons and toggles.

<CodeBlock
  code={'<button data-anim="bounce" data-anim-trigger="click">\n  Click me to bounce\n</button>'}
  lang="html"
/>

## Focus

Plays when the element receives focus. Ideal for form inputs.

<CodeBlock
  code={'<input\n  type="text"\n  data-anim="pulse"\n  data-anim-trigger="focus"\n  placeholder="Focus me"\n/>'}
  lang="html"
/>

## Combining Triggers

Each element can only have one trigger. Nest elements for multiple interactions:

<CodeBlock
  code={'<div data-anim="fadeInUp">\n  <button data-anim="pulse" data-anim-trigger="hover">\n    Interactive button\n  </button>\n</div>'}
  lang="html"
/>

## Summary

| Trigger | Value | Use Case |
|---------|-------|----------|
| Viewport | `viewport` (default) | Scroll reveals |
| Hover | `hover` | Buttons, cards |
| Click | `click` | Toggles, confirmations |
| Focus | `focus` | Form inputs |
