# Responsive

Not every animation makes sense on every screen size. data-anim provides responsive attributes to disable or swap animations based on viewport width.

## Disable Animations

Use `data-anim-disable` to turn off animations at specific breakpoints.

<AttributeTable attributes={[
  { name: 'data-anim-disable', type: '"mobile" | "tablet" | "desktop"', description: 'Disable the animation at the specified breakpoint.' },
]} />

<CodeBlock
  code={`<!-- No animation on mobile -->
<div data-anim="slideInLeft" data-anim-disable="mobile">
  This only animates on tablet and desktop
</div>

<!-- No animation on tablet -->
<div data-anim="fadeInUp" data-anim-disable="tablet">
  Skips animation on tablet
</div>`}
  lang="html"
/>

### Breakpoints

| Breakpoint | Range |
|-----------|-------|
| `mobile` | < 768px |
| `tablet` | 768px - 1023px |
| `desktop` | >= 1024px |

## Mobile Override

Use `data-anim-mobile` to swap to a different (typically simpler) animation on mobile.

<AttributeTable attributes={[
  { name: 'data-anim-mobile', type: 'string', description: 'Alternative animation name for mobile screens (< 768px).' },
]} />

<CodeBlock
  code={`<!-- Full slide on desktop, simple fade on mobile -->
<div data-anim="slideInLeft" data-anim-mobile="fadeIn">
  Responsive animation
</div>`}
  lang="html"
/>

This is useful when:
- Complex animations cause jank on low-end mobile devices
- Horizontal slide animations don't work well on narrow screens
- You want a more subtle experience on mobile

## Best Practices

### Prefer simpler animations on mobile

<CodeBlock
  code={`<!-- Good: simpler animation on mobile -->
<div data-anim="slideInLeft" data-anim-mobile="fadeIn">Content</div>

<!-- Good: disable unnecessary animation -->
<div data-anim="parallax" data-anim-disable="mobile">Content</div>`}
  lang="html"
/>

### Consider reduced motion

data-anim automatically respects the `prefers-reduced-motion` media query. When the user has requested reduced motion, all animations are disabled system-wide. You don't need to handle this manually.

### Performance on mobile

- Horizontal slides (`slideInLeft`, `slideInRight`) can cause horizontal overflow on mobile
- Zoom animations are GPU-friendly and work well on all devices
- Fade animations are the most performant choice for mobile
